    Gene set CNV & survival module provides a summary of survival between gene set CNV and wide type.

  • Methods

    • The clinical survival data and CNV data were downloaded from the TCGA database. And the CNV data and clinical survival data were merged by sample barcode. Samples that have a competing risk for death from cancer were filtered out (for DSS and DFI data).

    The gene set CNV represents the integrated CNV status of inputted gene set for each sample. A sample is classified into Amp. or Dele. group, only when at least one gene in the input gene set is consistently amplified or deleted in this sample. If all genes in inputted gene set have no CNV in a sample, this sample is classified into the WT group. When genes have inconsistent CNV status, for example, gene A was amplified in sample 1, gene B was deleted in sample 1, then sample 1 will be classified in the Excluded group, which will not be considered in this analysis.

    Survival analysis was performed when at least 2 groups with more than 2 samples. And groups with < 2 samples also remained to draw survival curves.

    R package survival was used to fit the survival time and survival status within gene set CNV groups. Logrank test was performed to test the survival difference between groups.

    There are some suggestions for survival analysis:

    • Data availability

      • The OS data are available for all cancer types.
      The PFS data are not available for LAML.
      The DSS data are not available for LAML.
      The DFI data are not available for LAML, SKCM, THYM, and UVM.

    • The survival analysis can be performed as long as there are survival events. However, there are a few things you should know about the endpoints of TCGA OS, PFI, DFI, and DSS data (Liu et al., Cell, 2018).

      • ACC: The number of events is small;
      BRCA: These samples need a longer follow-up for OS and DSS;
      CHOL: The sample size is too small for OS, DSS, DFI, and PFI;
      DLBC: The sample size and number of events are too small, and these events need a longer follow-up;
      GBM: The number of disease-free cases is small;
      KICH: The number of events is too small, and these events need a longer follow-up;
      KIRC: The number of events is small;
      LGG: These samples need a longer follow-up for OS and DSS;
      LIHC: These samples need a longer follow-up DSS;
      MESO: The sample size for DFI is small;
      PCPG: These samples need a longer follow-up for OS, DSS, DFI, and PFI. The number of events is small;
      PRAD: These samples need a longer follow-up for OS and DSS;
      READ: These samples need a longer follow-up for OS, DSS, and DFI. The number of events for DFI is too small;
      TGCT: The number of events is small for OS and DSS, and these events need a longer follow-up;
      THCA: The number of events is small for OS and DSS, and these events need a longer follow-up;
      THYM: The number of events is too small for OS and DSS, and these evens need a longer follow-up;
      UCS: The sample size is small.

  • Interpretation

    • GSCA provides two forms of results, including tables and figures.

    • Table:

      • The table presents detailed information about Cancer type, Survival type, and Logrank P value. Survival types include OS, PFS, DSS, and DFI.

    • GSCA provides result figures, including:
      • Kaplan-Meier curves:

        • By clicking any one button in the table, GSCA would draw the Kaplan-Meier curves between groups.

      • Bubble plot:

        • This plot presents the Logrank p value through bubble color and size. The column is selected cancer types and the row is survival types (OS, PFS, DSS, and DFI). The bubble color from blue to red represents the hazard ratio from low to high, bubble size is positively correlated with the significance of the Logrank P value.
