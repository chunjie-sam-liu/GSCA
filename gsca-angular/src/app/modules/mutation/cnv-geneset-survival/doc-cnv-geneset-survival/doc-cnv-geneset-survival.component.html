    Gene set CNV & survival module provides summary of survival between gene set CNV and wide type.

  • Methods

    • The clinical survival data and CNV data were downloaded from TCGA database. And the CNV data and clinical survival data were merged by sample barcode. Samples that have competing risk for death from cancer were filtered out (for DSS and DFI data).

    The gene set CNV represents the integrated CNV status of inputted gene set for each sample. A sample is classified into Amp. or Dele. group only when at least one gene in the input gene set is consistently amplified or deleted in this sample. If all genes in inputted gene set have no CNV in a sample, this sample is classified into WT group. When genes have inconsistent CNV status, for example, gene A was amplified in sample 1, gene B was deleted in sample 1, then sample 1 will be classified in Excluded group, which will not be considered in this analysis.

    Survival analysis was performed when at least 2 groups with more than 2 samples. And groups with < 2 samples were also remained to draw survival curve.

    R package survival was used to fit survival time and survival status within gene set CNV groups. Logrank test was performed to test the survival difference between groups.

    There are some suggestions for survival analysis:

    • Data availability

      • The OS data are available for all cancer types.
      The PFS data are not available for LAML.
      The DSS data are not available for LAML.
      The DFI data are not available for LAML, SKCM, THYM, and UVM.

    • Attention for using the endpoints of OS, PFI, DFI, and DSS (Liu et al, Cell, 2018).

      • ACC: number of events is small;
      BRCA: need a longer follow-up for OS and DSS;
      CHOL: sample size is too small for OS, DSS, DFI, and PFI;
      DLBC: sample size and number of events are too small, need a longer follow-up;
      GBM: number of disease-free cases is small;
      KICH: number of events is too small, need a longer follow-up;
      KIRC: number of events is small;
      LGG: need a longer follow-up for OS and DSS;
      LIHC: need a longer follow-up DSS;
      MESO: sample size for DFI is small;
      PCPG: need a longer follow-up for OS, DSS, DFI, and PFI; number of events is small;
      PRAD: need a longer follow-up for OS and DSS;
      READ: need a longer follow-up for OS, DSS, and DFI; number of events for DFI is too small;
      TGCT: number of events is small for OS and DSS; need a longer follow-up;
      THCA: number of events is small for OS and DSS; need a longer follow-up;
      THYM: number of events is too small for OS and DSS; need a longer follow-up;
      UCS: sample size is small.

  • Interpretation

    • GSCA provides two forms of results, including table and figure.

    • Table:

      • The table presents detailed information about Cancer type, Survival type, Logrank P value. Survival type including OS, PFS, DSS, and DFI.

    • GSCA provides result figures, including:
      • Kaplan-Meier curves:

        • By clicking buttons in table, GSCA draws Kaplan-Meier curves between groups in specific cancer.

      • Bubble plot:

        • Presents the Logrank p value through bubble color and size. The column is selected cancer types and the row is survival type (OS, PFS, DSS, and DFI). The bubble color from blue to red represents the hazard ratio from low to high, bubble size is positively correlate with the significance of Logrank P value.
